# Руководство по импорту схемы БД в DBforge for MySQL

## Шаг 1: Подключение к серверу MySQL

1. Откройте **DBforge for MySQL**
2. В левой панели нажмите на **"New Connection"** (или используйте меню)
3. Заполните следующие данные:

   | Поле | Значение |
   |------|---------|
   | **Connection Name** | Orders DB |
   | **Host** | 192.168.100.35 |
   | **Port** | 3306 |
   | **User** | user01 |
   | **Password** | 81328 |
   | **Default Database** | mysql |

4. Нажмите **"Test Connection"** для проверки
5. Если успешно, нажмите **"OK"**

## Шаг 2: Создание новой базы данных

1. В левой панели разверните подключение "Orders DB"
2. Нажмите правой кнопкой на папку **"Databases"**
3. Выберите **"New Database"**
4. Введите название: `orders_db`
5. Нажмите **"OK"**

## Шаг 3: Выполнение SQL скрипта

### Вариант 1: Через Query Editor

1. Откройте подключение "Orders DB" двойным кликом
2. В верхнем меню выберите **"File"** → **"Open SQL Script"**
3. Найдите файл `database_schema.sql` в папке проекта
4. Откройте файл
5. Нажмите **"Execute"** (или Ctrl+Enter)
6. Проверьте результат выполнения

### Вариант 2: Через копирование кода

1. Откройте файл `database_schema.sql` в текстовом редакторе
2. Скопируйте весь текст (Ctrl+A, Ctrl+C)
3. В DBforge нажмите **"New Query"**
4. Вставьте код (Ctrl+V)
5. Выделите весь текст
6. Нажмите **"Execute"** (или Ctrl+E)

## Шаг 4: Проверка создания таблиц

1. В левой панели разверните **"Orders DB"** → **"Tables"**
2. Вы должны увидеть следующие таблицы:
   - `users` - информация о клиентах
   - `pickup_points` - пункты выдачи
   - `products` - товары
   - `orders` - заказы
   - `order_items` - товары в заказах

3. Щелкните правой кнопкой на таблице и выберите **"View Data"** для проверки содержимого

## Шаг 5: Проверка данных

### Проверка пользователей
```sql
SELECT * FROM users;
```
Должны быть 3 пользователя (Иван, Мария, Петр)

### Проверка товаров
```sql
SELECT * FROM products;
```
Должны быть 5 товаров (ноутбук, клавиатура, мышь, монитор, наушники)

### Проверка пунктов выдачи
```sql
SELECT * FROM pickup_points;
```
Должны быть 3 пункта выдачи

## Возможные ошибки и решения

### Ошибка: "Access Denied for user 'user01'@'192.168.100.35'"

**Решение:**
1. Проверьте, что пользователь user01 существует
2. Выполните на сервере MySQL:
```sql
CREATE USER 'user01'@'%' IDENTIFIED BY '81328';
GRANT ALL PRIVILEGES ON *.* TO 'user01'@'%' WITH GRANT OPTION;
FLUSH PRIVILEGES;
```

### Ошибка: "Can't connect to MySQL server"

**Решение:**
1. Убедитесь, что MySQL сервис запущен
2. Проверьте правильность IP адреса (192.168.100.35)
3. Убедитесь, что порт 3306 открыт в файрволле

### Ошибка: "Database 'orders_db' already exists"

**Решение:**
1. Удалите существующую БД: 
```sql
DROP DATABASE orders_db;
```
2. Выполните скрипт снова

### Ошибка: "Table already exists"

**Решение:**
Скрипт использует `CREATE TABLE IF NOT EXISTS`, поэтому эта ошибка не должна появиться. Если появилась, выполните:
```sql
DROP DATABASE orders_db;
```
И начните с шага 2.

## Экспорт структуры БД

Если нужно сделать резервную копию:

1. Нажмите правой кнопкой на БД "orders_db"
2. Выберите **"Export"**
3. Выберите формат **"SQL Dump"**
4. Нажмите **"Export"** и выберите место сохранения

## Дополнительные команды MySQL

### Просмотр всех БД
```sql
SHOW DATABASES;
```

### Выбор БД
```sql
USE orders_db;
```

### Просмотр всех таблиц
```sql
SHOW TABLES;
```

### Просмотр структуры таблицы
```sql
DESCRIBE users;
```

### Просмотр индексов
```sql
SHOW INDEXES FROM orders;
```

### Отключение проверки иностранных ключей (если нужно удалить данные)
```sql
SET FOREIGN_KEY_CHECKS=0;
-- Ваши DELETE команды здесь
SET FOREIGN_KEY_CHECKS=1;
```
