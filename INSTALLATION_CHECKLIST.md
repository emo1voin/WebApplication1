## Что делать, если что-то не работает?

### Проблема: "Connection refused"
```
❌ Error: Cannot connect to 192.168.100.35:3306
```
**Решение:**
- [ ] Проверьте, что MySQL сервер запущен
- [ ] Проверьте адрес IP (192.168.100.35)
- [ ] Проверьте порт 3306 открыт в файрволе
- [ ] Проверьте учетные данные user01/81328

### Проблема: "Database not found"
```
❌ Unknown database 'orders_db'
```
**Решение:**
- [ ] Убедитесь, что БД создана в DBforge
- [ ] Выполните database_schema.sql снова
- [ ] Проверьте имя БД в appsettings.json

### Проблема: "SSL Certificate Error"
```
❌ CERTIFICATE_VERIFY_FAILED
```
**Решение:**
- [ ] В браузере нажмите "Advanced" → "Proceed"
- [ ] Это нормально для self-signed сертификатов
- [ ] В cURL используйте флаг `-k`

### Проблема: "Port 7147 in use"
```
❌ Error: Unable to start Kestrel. Address already in use.
```
**Решение:**
- [ ] Закройте другое приложение на порту 7147
- [ ] Или измените порт в launchSettings.json

### Проблема: "Dependency not found"
```
❌ error NU1101: Unable to find package
```
**Решение:**
- [ ] Выполните: `dotnet restore`
- [ ] Проверьте интернет соединение
- [ ] Очистите NuGet кеш: `dotnet nuget locals all --clear`

### Проблема: "Build failed"
```
❌ error CS0103: The name does not exist
```
**Решение:**
- [ ] Выполните: `dotnet clean`
- [ ] Выполните: `dotnet restore`
- [ ] Выполните: `dotnet build`
