# Система управления заказами

Полнофункциональное веб-приложение для управления заказами, включающее REST API на ASP.NET Core и web-интерфейс.

│   ├── OrdersController.cs     # Управление заказами
│   ├── ProductsController.cs   # Каталог товаров
│   ├── ClientsController.cs    # Список клиентов
│   └── PickupPointsController.cs # Пункты выдачи
├── Models/                      # Модели данных

## Использование

### Web Interface

1. Откройте https://localhost:7147 в браузере
2. Выберите вкладку **"Заказы"** для просмотра всех заказов
3. Выберите вкладку **"Создать заказ"** для создания нового заказа

#### Создание заказа:
- Выберите клиента из списка
- Выберите пункт выдачи
- Укажите дату доставки (опционально)
- Добавьте товары (нажимая "+ Добавить товар")
- Для каждого товара выберите наименование и количество
- Нажмите "Создать заказ"

#### Управление заказом:
- Нажмите на заказ для просмотра деталей
- Измените статус заказа (В ожидании → Обработка → Готово → Завершено)
- Отмените заказ при необходимости

### REST API

Полная документация доступна в файле `REST_API_DOCUMENTATION.md`

#### Основные endpoints:

**Создание заказа:**
```bash
POST /api/orders
```

**Получение всех заказов:**
```bash
GET /api/orders
```

**Получение заказа по ID:**
```bash
GET /api/orders/{id}
```

**Получение товаров:**
```bash
GET /api/products
```

**Получение клиентов:**
```bash
GET /api/clients
```

**Получение пунктов выдачи:**
```bash
GET /api/pickup-points
```

## Решение проблем

### Ошибка подключения к БД
- Проверьте, что MySQL сервер запущен на 192.168.100.35
- Убедитесь, что пользователь user01 с паролем 81328 существует
- Проверьте, что база данных `orders_db` создана

### Ошибка 503 Service Unavailable
- Убедитесь, что приложение запущено
- Проверьте, что порт 7147 не занят другим приложением
- Перезагрузите приложение

### API возвращает 404
- Проверьте URL запроса
- Убедитесь, что все зависимости установлены: `dotnet restore`
- Перестройте проект: `dotnet build`
